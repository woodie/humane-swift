# humane-swift v0.9.3

## One `Humane` enum, `distanceInTime`/`timeAgo` split

Still converging on the final API -- versioned `0.9.3`, not `1.0.0`, on
purpose. `TimeFormatter`/`SizeFormatter` merge into a single `enum Humane`
-- `.humanSize`, `.distanceInTime`, `.timeAgo`. The old two-argument
`.timeAgo` is renamed `.distanceInTime`; a new one-argument `.timeAgo`
takes its place. Same naming split as ActionView's own
`distance_of_time_in_words` (explicit two-input core) and
`time_ago_in_words` (one-input convenience wrapping it with `Time.now`) --
the one place in this family where Rails' naming, not Foundation's, drives
the shape (`RelativeDateTimeFormatter` has no built-in "now" convenience of
its own). `humane` (Go) and `humane-ruby` picked up the equivalent changes
in the same session -- see `docs/COWORK.md`'s `v0.9.3` entry for the full
rationale.

```swift
SizeFormatter.humanSize(225_935)                  // old
Humane.humanSize(225_935)                         // new -- "226 KB"

TimeFormatter.timeAgo(mtime, now)                  // old -- "3 minutes ago"

Humane.distanceInTime(mtime, now)                  // new -- explicit relativeTo, same output
Humane.timeAgo(mtime)                              // new -- one argument, relativeTo is Date()
```

- **No more `TimeFormatter`/`SizeFormatter` enums.** `.humanSize`,
  `.distanceInTime`, and `.timeAgo` are static functions on `Humane`
  directly.
- **`.distanceInTime`** is the explicit, fully-tested core -- unchanged
  behavior from the old `.timeAgo`, just renamed and un-namespaced. This is
  what specs should call.
- **`.timeAgo`** is a new one-argument convenience for the common "drop
  into a view" case. Supplies `Date()` as `relativeTo` internally; every
  parameter (`approximate:`, `includeSeconds:`, `whenNil:`) still applies.
- Not invented from scratch: `zouk`'s own `ScanEntry.timeAgo` (a
  zero-argument computed property wrapping `Date()`, sitting beside a
  `timeAgo(relativeTo:)` method for specs) had already hand-built this
  exact two-tier split, because this package didn't offer it. The new
  one-argument `Humane.timeAgo` here (and `humane`/`humane-ruby`'s matching
  convenience) lets consumers with a production-only call site drop their
  wrapper -- though `zouk`'s own `ScanEntry.timeAgo(relativeTo:)` should
  keep calling the explicit `distanceInTime`, not the new convenience,
  since it deliberately needs an injectable clock for its specs.

## Upgrading

Every call site needs a change:

```swift
// old
SizeFormatter.humanSize(size)
TimeFormatter.timeAgo(t, now)

// new
Humane.humanSize(size)
Humane.distanceInTime(t, now)
```

If the call site was only ever supplying `Date()` for `relativeTo` (the
common case in a view, not a test), simplify to the new one-argument form
instead:

```swift
// old
TimeFormatter.timeAgo(t, Date())

// new
Humane.timeAgo(t)
```

`zouk` is the only known Swift consumer and needs this follow-up pass --
see its own `docs/COWORK.md` once that happens. `ScanEntry.timeAgo(relativeTo:)`
should map to `Humane.distanceInTime`, not `Humane.timeAgo`, since it keeps
an explicit clock parameter for `ScanEntrySpec`.
