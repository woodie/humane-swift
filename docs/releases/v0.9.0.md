# humane-swift v0.9.0

## A full API rethink -- static functions, new defaults, nil-aware `timeAgo`

Breaking change throughout. Through `v0.3.0` this package mirrored
`ByteCountFormatter`/`RelativeDateTimeFormatter`'s own instantiated-formatter
API shape. With three real consumers now shipped, the bar changed: `humane`
(Go) and `humane-ruby` need to feel as simple to drop into a template as
ActionView's own helpers, and this package needed to move with them for
consistency. See `docs/COWORK.md`'s `v0.9.0` entry for the full rationale.
Versioned `0.9.0` (a jump from `v0.3.0`, matching the identical jump
`humane` and `humane-ruby` make in the same session) rather than `v0.4.0`
or `v1.0.0` -- this redesign is the most considered this API has ever been,
informed by three real consumers rather than a first guess, but `0.9.0`
treats it as a release-candidate for `v1.0.0` rather than a full stability
promise just yet.

```swift
SizeFormatter().string(fromByteCount: 225_935) // old
SizeFormatter.humanSize(225_935)                // new -- "226 KB"

TimeFormatter(approximate: true).string(for: t, relativeTo: now) // old
TimeFormatter.timeAgo(t, now)                                     // new -- "about 15 hours ago"
```

- **No more instantiation.** `SizeFormatter`/`TimeFormatter` are now
  case-less enums with static functions (`.humanSize`, `.timeAgo`) instead
  of types built with `.init()`. Configuration moves from initializer
  parameters to call-site parameters.
- **`SizeFormatter.humanSize` is unchanged internally** -- still a
  one-line `ByteCountFormatter` passthrough, still authoritative.
- **`timeAgo`'s `approximate` now defaults to `true`**, matching
  ActionView's own always-on-past-the-hour-boundary behavior, and matching
  what `zouk` already passed explicitly. `includeSeconds` stays `false` by
  default, unchanged.
- **`timeAgo` accepts `at: Date?`** and a new `whenNil:` parameter,
  returned as-is (no formatting) when `at` is `nil` -- lets a caller
  collapse a guard-then-fallback dance into one call.
- **`for:`/`at:`/positional overloads collapsed to one signature** --
  `timeAgo(_:_:approximate:includeSeconds:whenNil:)`. Those existed to
  bridge Foundation's own argument label and the `at` every other
  language was forced into; not needed once this package stopped
  mirroring `RelativeDateTimeFormatter`'s shape.

## Upgrading

Every call site needs a change:

```swift
// old
TimeFormatter().string(for: t, relativeTo: now)

// new
TimeFormatter.timeAgo(t, now)
```

If you were relying on `approximate: false` (the old default) for exact
wording rather than "about" phrasing, pass it explicitly now:

```swift
TimeFormatter.timeAgo(t, now, approximate: false)
```

`zouk` is the only known Swift consumer and needs this follow-up pass --
see its own `docs/COWORK.md` once that happens. `ScanEntry.timeAgo`'s
current `guard let downloadedAt else { return nil }` wrapper can likely be
replaced by calling `Humane.TimeFormatter.timeAgo` directly with
`whenNil:`, dropping the wrapper method entirely.
